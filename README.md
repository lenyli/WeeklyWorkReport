# 智慧云影研发部工作周报

这是一个带数据收集服务的工作周报应用。所有人必须访问同一个服务端地址，提交、汇总和 Excel 导出才能正常工作。

## 重要说明

GitHub Pages 只能发布静态网页，不能运行 `server.js`，因此不能用于自动收集周报。

`192.168.x.x` 这类地址是局域网地址，只能在同一个网络内访问。成员在不同地点时，需要部署到公网服务器，并使用公网 HTTPS 地址访问。

## 推荐访问方式

部署完成后，只使用公网服务地址，例如：

```text
https://your-domain.example.com/
```

不要从 GitHub Pages、文件直接打开、或其他静态网页地址进入。

## 本机测试

本机测试可以运行：

```powershell
$env:PORT="5174"; node server.js
```

然后访问：

```text
http://127.0.0.1:5174/
```

这个地址只适合开发测试，不适合分发给不同地点的成员。

## 公网部署要求

部署平台需要支持：

- Node.js 服务
- 持久化文件或数据库
- HTTPS 访问

当前版本会把数据保存在：

```text
data/store.json
```

如果部署平台的文件系统重启后会清空，需要换成带持久化磁盘的平台，或后续改成数据库存储。

## 使用

1. 打开统一服务地址。
2. 输入姓名进入。
3. 选择本周周一日期。
4. 填写周报内容。
5. 点击“提交”。

填写页面点击“下载个人 Excel”可下载当前人员的周报 Excel。

汇总页面选择周一日期后，点击“导出 Excel”可生成汇总文件。

## 文件说明

- `server.js`：服务端接口和数据保存
- `index.html`：页面结构
- `app.js`：页面交互和 Excel 导出
- `styles.css`：页面样式
- `manifest.webmanifest`：应用配置
- `sw.js`：清理旧 PWA 缓存
- `icons/`：应用图标
